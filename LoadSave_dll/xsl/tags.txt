<save> das übertag für einen speicherstand
 <object> ein gespeichertes objekt
  <id> die interne id des objektes (für die factory)
  <name> name des objekts
  <record> interne laufende nummer (jedes objekt hat ne eigene), auch jedes array
  <var> eine variable des objekts, es gibt:
   -immer einen <type>
   -immer einen <name>
   -manchmal einen <type_modifier>:
    -keiner: eine <value>
    -array:
     <record> für das html-design (am besten andere record-variable als die für die objekt-identifikation, z.b. array_0 etc.)
     <arr_column_indices> die obere zeile einer tabelle (nur fürs auge zum orientieren)
      <value> die zahlen für die zeile
     <arr_line> eine zeile der tabelle. erlaubt:
      <index> die zahl links neben der zeile
      <value>(s) die einträge des arrays in dieser zeile
      <var> (rekursiv ;)
   <object> ein sub-objekt, auch rekursiv
