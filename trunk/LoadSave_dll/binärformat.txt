die binärdateien des load/save-systems basisieren auf chunks, die jedoch auch rekursion unterstützen

da sie auch "unbekannte" daten speichern sollen, hat jedes element ähnlich XML ein anfangs- und ein endbyte
diese werden in nutzdaten natürlich escaped, damit keine fehler entstehen

auf das anfangsbyte folgen ein int für die ID und ein int für den modifizierer
(der im übrigen vom low-level nur zum markieren von arrays benutzt wird)
danach kommt noch ein int für die referenzierung. das wird gebraucht, wenn ein später in der datei erscheinendes objekt dasselbe ist.

falls es ein array ist, folgt ein int mit den dimensionen des arrays und darauf eine liste ints mit des größen der dimensionen

dann folgen die daten... (wie gesagt auch rekursiv, daher die ...)

und am ende das end-byte

grafisch sieht das ganze dann so aus:

CHUNK_START
	ID
	MOD
	REFID
	[ARR_DIMS]
	[ARR_DIM0_SIZE]
	[ARR_DIM1_SIZE]
	[...]
	[ARR_DIMN_SIZE]

	<data>
CHUNK_END

aber wen interessierts.. das resultat wird ja eh als xml abgespeichert ;)
